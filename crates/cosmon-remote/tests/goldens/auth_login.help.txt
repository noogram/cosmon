Run the PKCE manual-paste flow: `start → email → confirm`

Usage: cosmon-remote auth login [OPTIONS] --email <EMAIL>

Options:
      --email <EMAIL>
          

      --profile <PROFILE>
          Profile name (default: `default_profile` from config.toml, or `$COSMON_REMOTE_PROFILE`)

      --code <CODE>
          Pre-supply the authorization code instead of prompting (smoke-test / non-interactive harness)

      --json
          Render machine-readable JSON instead of human text where applicable

      --token <TOKEN>
          Bearer JWT (overrides `$COSMON_REMOTE_TOKEN`). When unset and an authenticated call is made, the CLI mints a token via the profile's `oidc_url`

  -h, --help
          Print help (see a summary with '-h')

WORKFLOW — login drives the three wire steps for you, in order:
  1. start     opens a PKCE session on the adapter
  2. email     submits your e-mail; the adapter returns a verification URL
  3. confirm   you open the URL, authorize, paste the code back

The pasted code travels only to your own adapter. Re-running login
replaces the previously posed credential. Inspect or abort an
in-flight session with `auth status <session_id>` / `auth logout
<session_id>`.
