# Conformance markers for the PUBLICLY SERVED install.sh.

# served at the public one-liner endpoint. This is the *semantic* floor of the
# drift check in scripts/release/check-install-drift.sh — it names, in human
# terms, WHICH capability a stale served copy has silently dropped, instead of
# only printing an anonymous diff.
#

#   1. the gnu→musl target fix had to be re-synced to the served copy by hand;
#   2. v0.2.0 shipped the cosmon-remote connector correctly (signed assets on
#      all four triples, client tarball verifiably carrying BOTH binaries), but
#      the SERVED installer had zero references to cosmon-remote — so a fresh
#      one-liner install placed `cs` and silently discarded the connector, and
#      the documented remote-connect workflow stayed injouable from a fresh
#      public install even though the release itself was correct.
#
# INVARIANT: every marker here must also be present in the canonical
# infra/install/install.sh. The checker asserts that too, so a marker can never
# rot into a claim the source itself no longer makes.
#
