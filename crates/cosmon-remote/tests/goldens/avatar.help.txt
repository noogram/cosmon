D-AVATAR instance lifecycle (drained from cs-thin by the fusion)

Usage: cosmon-remote avatar [OPTIONS] <COMMAND>

Commands:
  status      `GET /v1/avatar/{instance_id}/status` ‚Äî mould or avatar state
  incarnate   `POST /v1/avatar/{instance_id}/incarnate` ‚Äî bind moule‚Üíavatar
  grant       `POST /v1/avatar/{instance_id}/grant` ‚Äî bind a canal
  audit       `GET /v1/avatar/{instance_id}/audit` ‚Äî cicatrice + events
  mould-info  `GET /v1/avatar/{instance_id}/mould-info` ‚Äî pre-incarnation info
  help        Print this message or the help of the given subcommand(s)

Options:
      --profile <PROFILE>  Profile name (default: `default_profile` from config.toml, or `$COSMON_REMOTE_PROFILE`)
      --json               Render machine-readable JSON instead of human text where applicable
      --token <TOKEN>      Bearer JWT (overrides `$COSMON_REMOTE_TOKEN`). When unset and an authenticated call is made, the CLI mints a token via the profile's `oidc_url`
  -h, --help               Print help
