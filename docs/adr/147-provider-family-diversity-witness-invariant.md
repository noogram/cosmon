# ADR-147 — Provider-family diversity: the error-independence witness invariant

**Status:** Accepted (constitutional invariant)
**Date:** 2026-07-11
**Decider:** Noogram
**Source:** `task-20260711-f95b` (📐 decision), the C1 root of the
decomposition in `delib-20260711-f62a` (panel: torvalds · feynman · turing ·
buterin · dewey).
**Parent deliberation:** `delib-20260711-f62a` — synthesis §D-2, §S-1, §S-2.
**Feeds:** C3 (`provider-bias.toml` + downgrade lint — implements tier-a),
C2 (cross-provider reading-committee formula — consumes the invariant).

## Context

Showroom's cosmon-mechanisms survey asked whether cosmon can run a
**cross-provider reading committee** — a molecule generated by one model,
audited by a reader that is *not the same model* — on root and
security-stake work. The survey framed the diversity requirement as a
janis-style "axe jamais assis" line: *the committee must include ≥1
non-Claude reader*. The `delib-20260711-f62a` panel verified this against
the real code and returned two structural corrections that this ADR
promotes to a constitutional invariant.

### Why this is a NEW axis, not a restatement of P_external (S-2)

Cosmon already has three independence mechanisms, and all three enforce the
**same** kind of independence:

- **P_external** ([ADR-032](032-p-external-witness-axiom.md)) — the witness
  must sit *outside* the universe it attests.
- **`SameSessionRefusal`** ([ADR-085](085-stress-test-seal-mechanism.md) §3,
  `crates/cosmon-runtime/src/witness.rs`) — the witness must run in a
  *session distinct* from the tackler.
- **mindguard / `cs witness`** — independence is checked on
  *molecule / session* identity; the provider never enters the check.

Every one of these buys **channel / session independence**. None of them
buys **error independence**. A Claude witnessing a Claude is
channel-independent (different session, different tmux socket, a legitimate
`SameSessionRefusal` pass) yet **error-correlated**: the two share weights,
training distribution, and therefore *the same blind spots*. If the
generator hallucinates a plausible-but-wrong root cause because of a
systematic bias in its training, an independent session of the same model
tends to hallucinate *the same* wrong root cause — the refuter is an **echo,
not a witness** (buterin). Anchoring, sycophancy, and confirmation biases
are model-family properties, not session properties.

Provider-family error-independence is therefore a genuinely new axis of
independence, orthogonal to the channel/session axis the existing
mechanisms already cover. It is new constitutional ground, which is why it
warrants an ADR rather than a footnote to ADR-032.

### Why "≥1 non-Claude" is not enforceable as stated (D-2)

The survey's line is decidable but **gameable by a proxy-costume attack**
(turing + buterin, strong convergence). Cosmon dispatches a worker by
matching the *declared adapter string*, not by inspecting the endpoint that
string resolves to:

- `AdapterEntry` (`crates/cosmon-core/src/config.rs`, the struct at
  ~line 1231) has **no `family` field**. It carries `base_url`,
  `default_model`, `api_key_env`, `pane_signatures` — every field needed to
  point an `[adapters.openai]`-shaped seat at *any* HTTP endpoint, and none
  that records which model family actually answers there.
- Dispatch is a `match` on `adapter.as_str()` (the selector arm in
  `crates/cosmon-cli/src/cmd/tackle.rs`; the survey's line numbers drift by
  ~ten across edits — follow the symbol, `resolve_adapter_selection` /
  `spawn_openai_session`, not the number).

The struct's own docstring already narrates the exact hole: an
`openai`-named adapter with a `base_url` override aimed at xAI / Moonshot /
DeepSeek. Nothing in the type system stops that same override from fronting
a **Claude-compatible** endpoint. So a committee that *declares* a seat
named `openai` can, in fact, be a second Claude wearing an OpenAI costume.
A name check would bless it; the audit it was supposed to provide is
`A → A` disguised as `A → B`.

"Non-correlated" in the full statistical sense — *these two endpoints do not
share a failure distribution* — is a property over a distribution,
uncomputable from config (Rice-flavored, turing). The invariant cannot
promise that. It can promise the weaker, decidable floor: the seats resolve
to **distinct endpoints**, and — at the attested tier — that the family
behind each endpoint is bound to a token the audited party cannot mint.

### Who selects the committee is the real collusion question (S-1)

buterin's decisive observation: **Q1 (the diversity invariant) and Q4
(add-only config) are the same mechanism.** A diversity constraint is only
collusion-resistant if the audited worker **cannot edit it**. If the worker
that generated the molecule also picks its own refuter — or can lower its
own requirement — the fox appoints the inspector. The constraint must be
**exogenous to the party it constrains**: it lives in the add-only baseline
(C3's monotone-union schema) that the worker's per-molecule config can only
*add to*, never subtract from. This is why C1 (this ADR, the requirement
vocabulary) feeds C3 (the baseline that homes it): they are two faces of
one guarantee.

## Decision

Adopt the **Provider-family diversity invariant** as a constitutional rule
for cosmon reading committees:

> **Provider-family diversity (error-independence witness).** A
> cross-provider reading committee explicitly requested by its fleet or spore
> for a **root-stake or
> security-stake** molecule MUST include **≥1 reader whose *resolved*
> endpoint tuple `(adapter, base_url, model-family)` differs from the
> generator's family.** Distinctness is measured on the *resolved* endpoint,
> **never on the declared adapter name.** The diversity requirement lives in
> the add-only baseline the audited worker cannot edit; the worker's
> per-molecule config may raise the bar, never lower it.

The invariant is enforced in **two tiers**, deliberately separated because
they make different promises.

### Tier (a) — the cheap floor: distinct resolved-endpoint tuples

A `cs reconcile --check` lint asserts that the committee's seats resolve to
**distinct `(adapter, base_url, model-family)` tuples**. This is a pure
config-level, decidable check: two seats that resolve to the same endpoint
redden the dry-run. It is implemented in **C3**
(`provider-bias-toml-additive-config-downgrade-lint`) by cloning
`check_no_strong_config_default` (`crates/cosmon-cli/src/cmd/reconcile.rs`
~line 523, invoked from the `--check` path) into a
`check_no_profile_requirement_downgrade` sibling that compares **resolved
requirement-ids + endpoint tuples**, never config-section names.

What tier (a) buys: it makes the *trivial* collapse — two seats pointing at
the same URL, or the same model behind two labels — **visible and
attributable**. What it does **not** buy: it cannot tell whether the family
*string* the config declares matches the weights actually answering at
`base_url`. The `model-family` component of the tuple is, at this tier,
operator-declared config, not a verified fact. A motivated proxy-costume
survives tier (a) if the operator lies in the config.

### Tier (b) — the attested tier: `SameFamilyRefusal`

For real assurance, bind family to an **attested provider-identity token**
carried on the `cs witness` ledger. This is a `SameFamilyRefusal` guard,
the sibling of the existing `SameSessionRefusal`
(`crates/cosmon-runtime/src/witness.rs`, ADR-085 §3): where
`SameSessionRefusal` refuses an attestation whose `witness_id` matches the
tackler session, `SameFamilyRefusal` refuses one whose **attested family**
matches the generator's. The family label is not self-declared — it is
bound to an **operator-attested or oracle-fingerprinted token** on the
ledger, because *"which weights sit behind this endpoint"* is precisely the
claim P_external says the system **cannot self-verify**. The token comes
from outside; the ledger records it; the guard reads it.

`SameFamilyRefusal` is specified here but is an **ADR-grade follow-on to
implement** (it needs the ledger schema for a family token, which does not
exist today). Tier (a) ships with C3; tier (b) is the constitutional target
that tier (a) approximates.

### The §8b trace-visibility ceiling (explicit)

Both tiers inherit the **§8b ceiling** (`docs/architectural-invariants.md`
§8b — *propose mechanisms of verification, do not impose them*). The lint is
a **CI dry-run**: it is bypassable by `--no-verify`, by skipping the step,
by an operator who mislabels the family in config (tier a) or who mints a
false attestation (tier b — though tier b raises the cost to "forge a
ledger token" rather than "type a different string"). The invariant makes a
mono-family committee **loud and attributable, not impossible.** An honest
"makes it loud" beats a false "enforces." Nothing in this ADR is sold as a
lock.

### The residual: stake self-classification (S-3)

Elasticity is **conserved, not eliminated** — it migrates. The invariant
binds on *root-stake / security-stake* molecules. A worker never has to
*remove* the requirement to dodge it; it can simply **declare its own
molecule low-stake** so the committee requirement never triggers. A term a
self-interested party assigns to itself is elastic by nature, and **no
schema closes this** — not the monotone-union baseline (C3), not the
endpoint lint, not the attested token. The only thing that polices
stake-self-classification is the **empirical probe** (C5, the per-provider
calibration corpus): it re-measures whether a model's self-declared
low-stake diagnoses actually hold up under cross-provider refutation. This
ADR names the residual as *open by design*; it does not pretend it is
closed.

## Consequences

### Constitutional

- Provider-family error-independence is a distinct independence axis,
  orthogonal to P_external (ADR-032) and `SameSessionRefusal` (ADR-085 §3).
  A committee that passes `SameSessionRefusal` may still violate this
  invariant, and vice versa; both must hold on root/security-stake work.
- Diversity is measured on the **resolved endpoint**, never the declared
  adapter name. Any future mechanism that checks committee independence by
  adapter string is a structural breach — file a bead.

### Operational

- **C3 implements tier (a):** the resolved-`(adapter, base_url,
  model-family)`-distinctness lint under `cs reconcile --check`, homed in
  the add-only `provider-bias.toml` baseline (monotone-union schema, no
  subtract field). The diversity requirement is exogenous to the audited
  worker.
- **C2 consumes the invariant:** the explicitly opted-in cross-provider reading-committee
  formula (a DAG of adapter-pinned reader molecules reusing tackle-time
  Path A, *not* a deep-think panel, which is Claude-only by construction)
  reddens `cs reconcile --check` if it silently collapses to one family.
- **Tier (b) is a tracked follow-on:** `SameFamilyRefusal` needs a family
  token on the `cs witness` ledger. Until it lands, tier (a) is the floor
  and the `model-family` tuple component is operator-declared config, not a
  verified fact — this limitation must be stated wherever the lint's output
  is surfaced.

### Architectural

- `AdapterEntry` gains **no `family` field** as a consequence of this ADR.
  Family is *not* a static config attribute (that would re-introduce the
  proxy-costume: a lying string). It is either a **resolved tuple** (tier a,
  computed from `adapter` + `base_url` + `model`) or an **attested token**
  (tier b, external). Adding a self-declared `family: String` to
  `AdapterEntry` would be exactly the anti-pattern this ADR forbids.
- The invariant is a property of the **committee formula and the baseline
  config**, not a new runtime primitive. Consistent with C-2 of the
  synthesis: the cross-provider committee is a *formula* over existing
  bricks (the ADR-004 registry extension point), not reactor surgery.

## Rejected alternatives

- **"≥1 non-Claude by adapter-name."** The survey's line. Decidable but
  gameable by proxy-costume (an `openai`-named seat fronting Claude via
  `base_url`). Rejected as the enforcement mechanism; kept as the *intent*
  the resolved-endpoint tuple operationalizes honestly.
- **A self-declared `family: String` on `AdapterEntry`.** Re-introduces the
  exact hole — a config string the audited party controls. Rejected; family
  is resolved (tier a) or attested (tier b), never self-declared.
- **Full statistical "non-correlation" enforcement.** *"These two endpoints
  do not share a failure distribution"* is uncomputable from config
  (Rice-flavored). Rejected as unachievable; the resolved-endpoint floor +
  attested family are the decidable approximations that survive.
- **Enforcing diversity in the audited worker's own per-molecule config.**
  Lets the fox appoint the inspector (S-1). Rejected; the constraint lives
  in the add-only baseline exogenous to the worker.
- **Selling either tier as a hard lock.** Both inherit the §8b ceiling.
  Rejected on anti-laxisme grounds — an honest "makes it loud and
  attributable" is the correct claim.

## Open questions

- **Ledger schema for the tier-(b) family token** — what does an
  operator-attested / oracle-fingerprinted family token look like on the
  `cs witness` ledger, and how is it minted? Deferred to the
  `SameFamilyRefusal` implementation ADR.
- **`model-family` resolution for tier (a)** — the tuple needs a
  deterministic mapping from `(adapter, base_url, model)` to a family label
  for the distinctness comparison. C3 owns the concrete resolution rule
  (likely a small static table keyed on `base_url` host + `model` prefix,
  operator-overridable, with the standing caveat that the label is declared,
  not verified, until tier b).
- **Stake threshold** — exactly which molecules count as "root-stake /
  security-stake" is a stake-classification question, and per S-3 it is the
  residual elasticity this ADR cannot close. C5's empirical probe is the
  only policing mechanism.
