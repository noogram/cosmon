Initialise a new profile with just a host URL — the operator then sets `sub`, `aud`, `oidc-url` (or `install.sh` does it server-side via templating)

Usage: cosmon-remote config init [OPTIONS] <NAME> <HOST>

Arguments:
  <NAME>  Profile name (e.g. `tenant-demo-aws`, `local`)
  <HOST>  Base URL — preserve scheme as-is (`http://…` for loopback, `https://…` for Tailscale-served deployments)

Options:
      --profile <PROFILE>  Profile name (default: `default_profile` from config.toml, or `$COSMON_REMOTE_PROFILE`)
      --json               Render machine-readable JSON instead of human text where applicable
      --token <TOKEN>      Bearer JWT (overrides `$COSMON_REMOTE_TOKEN`). When unset and an authenticated call is made, the CLI mints a token via the profile's `oidc_url`
  -h, --help               Print help
