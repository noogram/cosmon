## Two corrections to the deliberation's premises

Both were discovered while executing this capture. Neither changes C0's assignment, but
each changes what a later child should believe.

### 1. `Trigger::Done` is not in the archive manifest, and the 917-manifest-read cost is not real

The synthesis states that `Trigger::Done` lives in the archive manifest
(`crates/cosmon-archive/src/lib.rs` L91-92) and that reading it would cost 917 per-row
manifest reads. Both halves are wrong on the actual data:

- **The manifest has no `trigger` field.** All 2426 on-disk `manifest.json` files carry
  `{formula_pin, molecule_id, response_hashes, schema_version, status}` and nothing else.
- **`cosmon-archive` is dead code.** It has zero callers outside its own crate. The
  `Manifest` struct that declares `pub trigger: Trigger` is never written to disk. The
  live writer is `crates/cosmon-state/src/archive/mod.rs`, which emits a *differently
  shaped* manifest.
- **The trigger is recorded — in `events.jsonl`,** one per archive entry, via
  `transition_event()` (`cosmon-state/src/archive/mod.rs` L323). Shape:
  `{"at":…,"molecule_id":…,"status":"completed","trigger":"done"}`.

So the falsifier is answerable, and cheaply — but from a different file than the panel
believed. **A `HARVEST` bucket would read `events.jsonl`, not manifests.** Whoever
prices that bucket should re-derive the cost rather than inherit the 917 figure.

### 2. Archive-side harvest lag is structurally zero and is the wrong place to look

Every archived `completed` entry carries `trigger: done` (1444 of 1444). This is not
evidence of health — archiving *happens at* `cs done`, so the archive cannot contain an
unharvested molecule by construction. The lag lives entirely in the **live** state,
which is why the number above is computed there. A future reader measuring lag from the
archive will find 0 and conclude, wrongly, that there is nothing to see.

Full trigger×status distribution across the archive (2422 entries, 0 missing
`events.jsonl`):

|---|---|---|
| completed | done | 1444 |
| collapsed | collapse | 883 |
| frozen | stuck | 108 |
| collapsed | done | 13 |
| running | freeze | 7 |
| pending | done | 2 |

The last three rows are status/trigger pairs that should not exist (a `running` molecule
is not terminal; a `pending` one was never worked). 22 entries, 0.9%. Out of scope for
C0 — recorded here because the baseline is the only place it will be visible.

## Noise floor — read this before diffing two snapshots

`cs peek --snapshot` is documented as byte-deterministic: *"the same fleet state
produces byte-identical output across every device."* That contract holds. But two
captures taken seconds apart on a **live** fleet differ anyway, because the fleet state
itself moves. Observed between two back-to-back captures during this task:

- **`ENERGY` and cost columns** advance continuously for every active worker.
- **The capturing worker's own row** is in the snapshot it captures (observer effect).
- **Sibling molecules transition mid-capture** — `task-20260515-50a2` went
  `completed` → `collapsed` between the two runs.

A before/after diff for the redesign is therefore **only meaningful on the
status/composition columns**, and only when read against these three sources of
movement. The determinism contract is about the renderer, not about a fleet that holds
still.

## How this number was computed

```sh
cd <galaxy-root>
ARCH=.cosmon/state/archive
M=.cosmon/state/fleets/default/molecules

# ids archived with a `done` trigger
find $ARCH -name events.jsonl ! -path "*_quarantine*" -exec cat {} \; \
  | jq -r 'select(.trigger=="done") | .molecule_id' | sort -u > /tmp/done_ids.txt

# live `completed` molecules with no `done` trigger == harvest lag
lag=0
for d in $M/*/; do
  id=$(basename "$d")
  [ "$(jq -r '.status // ""' "$d/state.json")" = "completed" ] || continue
  grep -qxF "$id" /tmp/done_ids.txt || lag=$((lag+1))
done
echo "HARVEST LAG = $lag"
```

Excludes `.cosmon/state/archive/_quarantine/` (4 entries) throughout.
